Execution Screen — Design Notes
  • 1
    Order Type Toggle — Segmented control appears only for Real Stocks. CFD instruments show legacy flow unchanged. Toggle state persists per instrument.
  • 2
    TLO Info Banner — Contextual explainer appears when "True Limit Order" tab is selected. Key facts: rests in market, no slippage, ticket fee, DAY order (expires EOD).
  • 3
    Market Data Strip — Live Ask/Bid/Spread/Tick Size helps user set a realistic price. Tick size is configurable per instrument via admin.
  • 4
    Limit Price Input — Dollar amount. Quick shortcuts relative to current ask (e.g. -1%, -2%). Validates against tick size. For a BUY: price above ask+tolerance triggers block. For SELL: inverse asymmetry logic applies.
  • 5
    Whole Shares Only — Fractional shares are disallowed. Dollar-amount entry rounds down to closest whole share. Sub-1-share amounts blocked at submission.
  • R
    Balance Reserved at Limit Price — Available balance check uses limit price × quantity (not current market price). Reserve shown inline before confirmation.
  • R
    Price Validation Asymmetry — Buy limit above ask → block (configurable %). Buy limit below ask → no error. Sell logic is inverse. Tolerance % is configurable per instrument.
  • !
    Out of scope — No SL/TP, no markups, no fractional shares, no amendments after submission, no CFDs. Copy trading (PI) follows separate flow.

Portfolio Screen — Design Notes
  • 1
    Status Chips — PENDING (pulsing dot), PARTIAL (blue), FILLED (green), CANCELLED (grey). Animated dot draws attention to active orders without being intrusive.
  • 2
    Fill Progress Bar — Shows "X of Y shares" with a color-coded progress bar. PARTIAL = blue bar. PENDING with 0 fills = thin amber sliver at left. 100% = full green.
  • 3
    TLO Badge — Small "TLO" chip on every limit order card distinguishes it from regular market positions at a glance.
  • 4
    Cancel Button — Visible only on PENDING and PARTIAL orders (manual submissions). Tapping triggers the Cancel Confirmation Sheet. Hidden on FILLED/CANCELLED positions.
  • 5
    Cancel Confirmation Sheet — Bottom sheet shows full order detail, reserved balance, and the edge-case warning (simultaneous fill + cancel = trade still goes through, CS alert triggered).
  • 6
    EOD Expiry — Position detail shows expiry time clearly ("Expires: EOD · 16:00 ET"). Unfilled shares at EOD are auto-cancelled; any partial fills are settled normally.
  • R
    Partial Fill Distribution — For Copy: partial fills are distributed throughout the copy tree as they occur (not just at terminal state). Same as existing copy logic.
  • !
    Dealing Desk filter — Limit order rejections must be hidden from the Dealing Desk rejection screen (DEAL-8825). Most limit orders fail by design; dealers should not see these.
